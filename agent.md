# Agent 工作规范

## 修改文件前检查

在修改任何文件之前，必须先检查 git 是否有未提交的修改：

1. 运行 `git status` 查看是否有未提交的更改
2. 如果有未提交的修改，必须先进行提交再进行其他修改
3. 提交信息应该清晰描述更改内容

## 提交规范

- 使用清晰的提交信息，说明修改的目的和内容
- 确保提交信息简洁明了

## 自动提交策略

启用 auto-commit skill，在每次代码修改后自动进行提交。

### 提交规则

1. **分步提交**: 如果工作有多步骤，就分步提交，每一步完成一个独立的变更后立即提交
2. **单一意图**: 每次提交只包含一个独立的变更意图
3. **即时提交**: 完成验证后立即提交，不等待用户提示

### 提交流程

1. 选择下一个独立的变更
2. 只实现该范围内的修改
3. 运行至少一个检查：针对性测试、lint/typecheck 或 build
4. 如果检查失败，修复问题后重新运行检查
5. 只暂存相关文件（使用 `git add <path...>`）
6. 提交包含意图特定的消息
7. 重复以上步骤

### 注意事项

- 避免广泛暂存
- 永远不要包含不相关的脏树变更
- 保持依赖优先的提交顺序（先提交前置依赖，再提交依赖它的变更）
